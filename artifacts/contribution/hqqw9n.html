代码贡献

1. 了解“隐语”开源项目

  • 若您还没有很了解“隐语”，欢迎到“隐语”官网，了解“隐语”框架中的相关概念、技术路线、框架设计思想、密态计算设备等。

2. 登陆 GitHub

  • 熟悉“隐语”代码仓库内容

3. 阅读 CLA 协议

  • 请详细查看 CLA 协议 内容，在提交 Pull Request 的过程中，将通过 CLA Assistant 自动触发个人 CLA 签署流程，完成签署后才能进入下一步流程。如您拒绝签署该协议，将无法继续后续操作。

如果是企业贡献，需要遵循企业 CLA 签署流程，具体可点击查看

4. 环境准备

  • 在正式开始代码贡献之前，您可以参照如下内容完成对开发环境的准备：
git clone https://github.com/secretflow/secretflow.git
conda create -n secretflow python=3.8
conda activate secretflow
pip install -r dev-requirements.txt -r requirements.txt

若您没有安装 miniconda，请点击此链接安装 conda。

5. 代码规范

  • 我们推荐使用 Black 为您贡献的代码进行格式规范，具体使用请参照 Editor integration；
  • 注意添加 -S, --skip-string-normalization 以防代码中的行引号及前缀被规范。

6. 提交 Pull Request（PR）

  • 如果您对“隐语”进行了改进，“码上”通过 PR 和我们分享吧！
    • 提交 PR 时，记得关联相关 Issue；
    • 如何没有找到合适的 Issue，也可以自己创建一个新的 Issue，并将 Issue 分配给自己。
  • 新朋友可以通过 GitHub 的“how to”指南了解提交 Pull Request 的方法。
  • 🎖贡献新功能时包揽了单元测试是社区中值得嘉奖的行为！
  • 隐语社区 Maintainer 与 Committer 将对您提交的 Pull Request 进行 review，一旦您的 Pull Request 获得批准并经过了集成测试，团队成员就会为您的修改添加“ready to pull”的标签，也就是我们正在尝试将您的请求提交至内部仓库，通过后您的 Pull Request 就会自动合并至相关仓库中。

加入特殊兴趣小组（SIG）

欢迎选择您感兴趣的 SIG 小组，并参与到 SIG 小组的日常建设中。

资质要求

（至少满足以下要求中的任意 1 项即可申请）

  • 隐私计算领域技术专家，有相关方向研究成果
  • 有将隐语应用于实际业务项目的经验
  • 将隐语应用于比赛、科研论文、书籍撰写、课程开设等。如：科研人员/高校教授

评审标准

获得所属 SIG 小组成员超半数赞成票，则审核通过

申请流程

  • 填写并投递申请至官方邮箱（secretflow-contact@service.alipay.com） 点击下载《隐语开源社区SIG组创建申请表》
  • SIG 组内部评定审核
  • 获取结果回执
  • 线上签约
  • 加入社区


参与社区活动

  • 加入“隐语 SecretFlow”微信交流群，讨论分享隐私计算相关技术

与社区共同成长

  • 社区不同角色对应不同的责任与权利，每种角色都是社区不可或缺的一部分。
  • 您可以通过参与共建不断积累经验和影响力，并获得角色上的成长。
  • 详细角色说明、责任权利、成长机制请查看隐语社区组织。

行为准则

“隐语”社区遵守开源社区《贡献者公约》V1.4中规定的行为守则，请参考V1.4版本。

如需举报侮辱、骚扰或其他不可接受的行为，您可以致信官方邮箱secretflow-contact@service.alipay.com，联系委员会处理。

贡献者们的承诺

为建设开放友好的环境，我们贡献者和维护者承诺：不论年龄、体型、身体健全与否、民族、性征、性别认同与表征、经验水平、教育程度、社会地位、国籍、相貌、种族、信仰、性取向，我们项目和社区的参与者皆免于骚扰。

我们的准则

有助于创造积极环境的行为包括但不限于：

      • 措辞友好且包容
      • 尊重不同的观点和经验
      • 耐心接受有益批评
      • 关注对社区最有利的事情
      • 与社区其他成员友善相处

参与者不应采取的行为包括但不限于：

      • 发布与性有关的言论或图像、不受欢迎地献殷勤
      • 捣乱/煽动/造谣行为、侮辱/贬损的评论、人身及政治攻击
      • 公开或私下骚扰
      • 未经明确授权便发布他人的资料，如住址、电子邮箱等
      • 其他有理由认定为违反职业操守的不当行为

我们的义务

      • 项目维护者有义务诠释何谓“妥当行为”，并妥善公正地纠正已发生的不当行为。
      • 项目维护者有权利和义务去删除、编辑、拒绝违背本行为标准的评论（comments）、提交（commits）、代码、wiki 编辑、问题（issues）等贡献；项目维护者可暂时或永久地封禁任何他们认为行为不当、威胁、冒犯、有害的参与者。

适用范围

      • 本行为标准适用于本项目。当有人代表本项目或本社区时，本标准亦适用于此人所处的公共平台。
      • 代表本项目或本社区的情形包括但不限于：使用项目的官方电子邮件、通过官方媒体账号发布消息、作为指定代表参与在线或线下活动等。
      • 代表本项目的行为可由项目维护者进一步定义及解释。

贯彻落实

      • 可以致信，向项目团队举报滥用、骚扰及不当行为。
      • 维护团队将审议并调查全部投诉，妥善地予以必要的回应。项目团队有义务保密举报者信息。具体执行方针或将另行发布。
      • 未切实遵守或执行本行为标准的项目维护人员，经项目负责人或其他成员决议，可能被暂时或永久地剥夺参与本项目的资格。
