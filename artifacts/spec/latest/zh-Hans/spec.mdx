#### TableSchema

表结构。

- 一个col必须属于 `id | feature | label` 。 默认为 feature 。
- 所有名字需要满足正则表达式 `[A-Za-z0-9.][A-Za-z0-9_>./]*` 。
- 所有数据类型须为

+ int8
+ int16
+ int32
+ int64
+ uint8
+ uint16
+ uint32
+ uint64
+ float16
+ float32
+ float64
+ bool
+ int
+ float
+ str

#### IoDef.TableAttrDef

一个表单的额外属性。

如果在一个 IoDef 中提供， 例如

```json
{
  "name": "feature",
  "types": [
      "int",
      "float"
  ],
  "col_min_cnt_inclusive": 1,
  "col_max_cnt": 3,
  "attrs": [
      {
          "name": "bucket_size",
          "type": "AT_INT"
      }
  ]
}
```

以为着一个用户提供了一个表作为 IO, 他仍然需要制定一些列作为 “feature”:

- col\_min\_cnt\_inclusive 为1: 至少选择一列。
- col\_max\_cnt\_inclusive 为3: 至多选择3列。并且每选择一列之后， 用户还需要设置一个int属性bucket\_size 。
