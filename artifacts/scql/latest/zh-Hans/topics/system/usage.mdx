# SCQL 使用流程

The flowchart shows how to use SCQL system, which contains two stages: prepare and query. In the prepare stage, the system is set up, and the necessary tables are created and initialized. In the query stage, queries are submitted, processed, and results are returned.

<Container type="note">
  All requests from user to SCDB can be submitted through [<span>synchronous api</span>](../../reference/http-api.mdx#sync-api) or [<span>asynchronous api</span>](../../reference/http-api.mdx#async-api).
</Container>

## Participants

- Alice: the user in party alice
- Bob: the user in party bob
- SCDB: the SCDB server

<Container type="note">
  Before the start of the project, Alice and Bob need to obtain the root user’s username and password from the admin.
</Container>

:target{#prepare-stage}

## Prepare stage

<DefinitionList>
  <dl>
    <dt>
      <DefinitionList.Term><span>1.0 create database</span></DefinitionList.Term>
    </dt>

    <dd>
      Alice or Bob creates a new database using the root user account through [<span>CREATE DATABASE Statement</span>](../../reference/lang/manual.mdx#create-database-stm), which can be considered a new project, SCQL supports running multiple projects simultaneously.
    </dd>

    <dt>
      <DefinitionList.Term><span>2.0 create and grant user</span></DefinitionList.Term>
    </dt>

    <dd>
      Alice creates user account “Alice” in SCDB using the root user account by [<span>CREATE USER Statement</span>](../../reference/lang/manual.mdx#create-user-stm), and grants access to the relevant database by [<span>GRANT Statement</span>](../../reference/lang/manual.mdx#scql-grant-revoke)
    </dd>

    <dt>
      <DefinitionList.Term><span>3.0 create and grant user</span></DefinitionList.Term>
    </dt>

    <dd>
      Bob creates user account “Bob” in SCDB using the root user account by [<span>CREATE USER Statement</span>](../../reference/lang/manual.mdx#create-user-stm), and grants access to the relevant database by [<span>GRANT Statement</span>](../../reference/lang/manual.mdx#scql-grant-revoke)
    </dd>

    <dt>
      <DefinitionList.Term><span>4.0 create table</span></DefinitionList.Term>
    </dt>

    <dd>
      Alice creates table in SCDB using the account “Alice” by [<span>CREATE TABLE Statement</span>](../../reference/lang/manual.mdx#create-table).
    </dd>

    <dt>
      <DefinitionList.Term><span>5.0 create table</span></DefinitionList.Term>
    </dt>

    <dd>
      Bob creates table in SCDB using the account “Bob” by [<span>CREATE TABLE Statement</span>](../../reference/lang/manual.mdx#create-table).
    </dd>

    <dt>
      <DefinitionList.Term><span>6.0 grant CCL</span></DefinitionList.Term>
    </dt>

    <dd>
      Alice grants CCL about the table created by Alice for Alice and Bob in SCDB using the account “Alice”, see [<span>SCQL CCL</span>](../ccl/intro.mdx) for more information.
    </dd>

    <dt>
      <DefinitionList.Term><span>7.0 grant CCL</span></DefinitionList.Term>
    </dt>

    <dd>
      Bob grants CCL about the table created by Bob for Alice and Bob in SCDB using the account “Bob”, see [<span>SCQL CCL</span>](../ccl/intro.mdx) for more information.
    </dd>
  </dl>
</DefinitionList>

:target{#execute-stage}

## Execute stage

<DefinitionList>
  <dl>
    <dt>
      <DefinitionList.Term><span>8.0 submit a DQL</span></DefinitionList.Term>
    </dt>

    <dd>
      Alice submits a DQL to SCDB through [<span>public/submit\_query</span>](../../reference/http-api.mdx#submit-query) or [<span>public/submit\_and\_get</span>](../../reference/http-api.mdx#submit-and-get) using the account “Alice”.
    </dd>

    <dt>
      <DefinitionList.Term><span>9.0 process query</span></DefinitionList.Term>
    </dt>

    <dd>
      SCDB will check ccl and execute query with engines.
    </dd>

    <dt>
      <DefinitionList.Term><span>10.0 return result</span></DefinitionList.Term>
    </dt>

    <dd>
      SCDB return the query result to Alice.
    </dd>
  </dl>
</DefinitionList>
