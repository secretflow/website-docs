    Federated dataframe holds <cite>vertical</cite> partitioned data.

    This dataframe is design to provide a federated pandas dataframe
    and just same as using pandas. The original data is still stored
    locally in the data holder and is not transmitted out of the domain
    during all the methods execution.

    The method with a prefix <cite>partition\_</cite> will return a dict
    \{pyu of partition: result of partition}.

  <Outline.Content>
    Read a comma-separated values (csv) file into VDataFrame.

    When specifying spu and keys, the fields specified by keys are used for PSI
    alignment.Fields used for alignment must be common to all parties, and other
    fields cannot be repeated across parties. The data for each party is
    supposed pre-aligned if not specifying spu and keys.

    <FieldList>
      <dl>
        <dt className="field-name">

        <dd className="field-body">
          - <Outline.ParameterTarget>filepath</Outline.ParameterTarget> –&#x20;

            The file path of each party. It can be a local file with a
            relative or absolute path, or a remote file starting with <cite>oss\://</cite>,
            <cite>http(s)://</cite>, E.g.
            ```python
            {
                PYU('alice'): 'alice.csv',
                PYU('bob'): 'bob.csv'
            }
            ```
          - <Outline.ParameterTarget>delimiter</Outline.ParameterTarget> – the file separator.
          - <Outline.ParameterTarget>dtypes</Outline.ParameterTarget> –&#x20;

            Participant field type. It will be inferred from the file if
            not specified, E.g.
            ```python
            {
                PYU('alice'): {'uid': np.str, 'age': np.int32},
                PYU('bob'): {'uid': np.str, 'score': np.float32}
            }
            ```
          - <Outline.ParameterTarget>spu</Outline.ParameterTarget> – SPU device, used for PSI data alignment.
            The data of all parties are supposed pre-aligned if not specified.
          - <Outline.ParameterTarget>keys</Outline.ParameterTarget> – The field used for psi, which can be single or multiple fields.
            This parameter is required when spu is specified.
          - <Outline.ParameterTarget>drop\_keys</Outline.ParameterTarget> – keys to removed, which can be single or multiple fields.
            This parameter is required when spu is specified since VDataFrame
            doesn’t allow duplicate column names.
          - <Outline.ParameterTarget>psi\_protocl</Outline.ParameterTarget> – Specified protocol for PSI. Default ‘KKRT\_PSI\_2PC’ for 2
            parties, ‘ECDH\_PSI\_3PC’ for 3 parties.
