# <Target id="id1" />组件

隐语组件基于两个标准。首先我们遵守 [隐语开放标准](https://www.secretflow.org.cn/docs/spec/latest) 。并且我们通过扩展标准定义了集群定义和其他数据类型，请查看 [扩展标准](comp_spec.mdx) 。扩展标准仅适用于隐语，并不适用于隐语生态下的其他隐私保护应用。对于扩展标准的详细解释在 [扩展标准注释](comp_spec_design.mdx) 。

我们将一些常用的SecretFlow 应用程序包装为组件。SecretFlow 组件列表不是最终版本，我们正在不断更新列表。完整的组件列表在 [SecretFlow 组件列表](comp_list.mdx) 。

现在除了在Python程序中引入lib, 还还可以使用组件API或者CLI调用隐语程序，请查阅 [SecretFlow 组件指南](comp_guide.mdx) 。

:target{#migration-to-secretflow-open-specification}

## 迁移至 [隐语开放标准](https://www.secretflow.org.cn/docs/spec/latest)

引入 <strong>隐语开放标准</strong> 导致了一些不兼容的改动，包括

:target{#comp-proto}

### comp.proto

1. <em>comp.proto</em> 重命名为 <em>component.proto</em>
2. 在 message <em>AttrType</em> 中， <em>AT\_UNDEFINED</em> 替换为 <em>ATTR\_TYPE\_UNSPECIFIED</em>
3. 在 message <em>Attribute</em> 中， <em>has\_lower\_bound</em> 重命名为 <em>lower\_bound\_enabled</em>， <em>has\_upper\_bound</em> 重命名为 <em>upper\_bound\_enabled</em>
4. 在 message <em>IoDef.TableAttrDef</em> 中， <em>attrs</em> 重命名为 <em>extra\_attrs</em>

:target{#data-proto}

### data.proto

1. 在 message <em>SystemInfo</em>, <em>app\_name</em> 重命名为 <em>app</em>， <em>secretflow</em> (SFClusterDesc) 替换为 <em>app\_meta</em> (Any)
2. Message <em>StorageConfig</em> 迁移至 <em>cluster.proto</em>
3. 在 message <em>IndividualTable</em> 和 <em>VerticalTable</em>， <em>num\_lines</em> 重命名为 <em>line\_count</em> 。
4. 在 message <em>DistData</em>， <em>sys\_info</em> 重命名为 <em>system\_info</em> 。

我们对带来的不便感到抱歉。

:target{#announcement}

## 公告

:target{#october-2023}

### October, 2023

1. 我们正式发布了 [隐语开放标准](https://www.secretflow.org.cn/docs/spec/latest) ， 包含了所有隐私计算应用共享的部分。
2. 剩余部分被称为 <strong>扩展标准</strong> 。

:target{#july-2023}

### July, 2023

从 SecretFlow 1.0.0 开始，我们正式推出基于组件规范的第一个版本和相应的 SecretFlow 组件列表。
