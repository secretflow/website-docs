                > arguments, and <cite>inputs</cite> cannot be provided via the default value
                > of a keyword argument.
                - NumPy array or Python scalar values in <cite>inputs</cite> get cast as
                  tensors.
                - Keras mask metadata is only collected from <cite>inputs</cite>.

                  using shape info from <cite>inputs</cite> only.
                - <cite>input\_spec</cite> compatibility is only checked against <cite>inputs</cite>.
                - Mixed precision input casting is only applied to <cite>inputs</cite>.
                  If a layer has tensor arguments in <cite>\*args</cite> or <cite>\*\*kwargs</cite>, their
                  casting behavior in mixed precision should be handled manually.
                - The SavedModel input specification is generated using <cite>inputs</cite>
                  only.
                - Integration with various ecosystem packages like TFMOT, TFLite,
                  TF.js, etc is only supported for <cite>inputs</cite> and not for tensors in
                  positional and keyword arguments.

                this is not recommended, for the reasons above.

                The following optional keyword arguments are reserved:
                \- <cite>training</cite>: Boolean scalar tensor of Python boolean indicating

                - <cite>mask</cite>: Boolean input mask. If the layer’s <cite>call()</cite> method takes a

                  generated for <cite>inputs</cite> by the previous layer (if <cite>input</cite> did come
